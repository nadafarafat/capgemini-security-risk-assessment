## Overview

Enterprise security risk assessment for Capgemini comparing with a Hypothetical Government Agency (HGA). Conducted comprehensive threat-vulnerability analysis, designed defense-in-depth architecture, and developed optimized risk mitigation strategies with ROI analysis.

**Course:** CY 5200 - Security Risk Management and Assessment  
**Professor:** Themis Papageorge (PhD MIT, Former VP at Guardium)  
**Institution:** Northeastern University  
**Author:** Mohammed Arafat Nadaf

## Project Scope

### Part A: HGA Risk Assessment
- Information system categorization (FIPS 199)
- Asset valuation and threat identification
- Threat-vulnerability pair analysis with probability mapping
- Current vs proposed security controls
- Risk prevention and response strategies

### Part B: Capgemini Implementation Plan
- Access control implementation
- Network infrastructure security
- Database security controls
- Application development security
- Wireless security measures
- Cybersecurity workforce planning

### Part C: Recommendations & ROI Analysis
- Prevention vs response control comparison
- Cost-benefit analysis
- Budget optimization strategies

## Threat Analysis

| ID | Threat | Description |
|----|--------|-------------|
| T1 | Payroll Fraud | False timesheets, fabricated employees, embezzlement |
| T2 | Payroll Errors | Data entry failures, coordination mistakes |
| T3 | Operations Interruption | Power outages, hardware failures, malicious actions |
| T4 | Information Disclosure | Unauthorized access to sensitive data |
| T5 | Network-related | External attacks, data tampering, eavesdropping |
| T6 | Natural Disasters | Floods, storms, infrastructure damage |
